# HotPulse - 需求文档

> 版本：v1.0 | 日期：2026-04-22 | 状态：已确认

---

## 一、产品概述

HotPulse 是一个 **自动化热点监控与通知系统**，帮助用户第一时间获取指定领域的热点动态（如 AI 大模型更新、科技新闻等），无需人工手动搜索。

---

## 二、功能需求

### 2.1 关键词监控（Keyword Monitor）

| 编号 | 功能点 | 说明 |
|------|--------|------|
| F01 | 添加监控关键词 | 用户手动输入关键词（如"Claude 4"、"GPT-5"） |
| F02 | 关键词管理 | 查看、启用/停用、删除关键词 |
| F03 | 自动轮询检查 | 每 30 分钟自动从多源搜索关键词相关内容 |
| F04 | AI 真实性验证 | 用 OpenRouter AI 判断内容是否真实相关（过滤假冒/无关内容） |
| F05 | 触发通知 | 发现真实相关内容后立即推送三路通知 |
| F06 | 触发历史 | 记录每次关键词触发的完整历史 |

### 2.2 热点自动发现（Topic Discovery）

| 编号 | 功能点 | 说明 |
|------|--------|------|
| F07 | 添加监控领域 | 用户输入领域范围（如"AI编程"、"量子计算"） |
| F08 | 定时热点采集 | 每 30 分钟从多源自动聚合该领域热点 |
| F09 | AI 摘要与排序 | OpenRouter AI 生成摘要、评分热度、去重 |
| F10 | 热点 Feed | 展示带摘要、来源、热度分的热点列表 |
| F11 | 时间轴视图 | 按时间展示热点演变 |

### 2.3 通知系统（Notification）

| 编号 | 功能点 | 说明 |
|------|--------|------|
| F12 | WebSocket 实时推送 | 浏览器长连接，页面实时收到通知 |
| F13 | 浏览器原生通知 | 系统级 Notification API，页面不在前台也能收到 |
| F14 | 邮件通知 | 通过 SMTP 发送 HTML 格式邮件（支持 Gmail/自定义） |
| F15 | 通知历史 | 所有通知记录，支持已读/未读状态 |

### 2.4 配置管理（Settings）

| 编号 | 功能点 | 说明 |
|------|--------|------|
| F16 | OpenRouter API Key | 配置 AI 服务密钥 |
| F17 | twitterapi.io Key | 配置 Twitter/X 数据源密钥 |
| F18 | 邮件 SMTP 配置 | 配置发件服务器（host/port/user/pass）和收件邮箱 |
| F19 | 数据源开关 | 独立开关各数据源（Twitter/GoogleNews/HN/Reddit/GitHub/arXiv） |
| F20 | 检查频率 | 默认 30 分钟，可配置 |

---

## 三、非功能需求

| 类别 | 要求 |
|------|------|
| 响应式 | 支持 375px（手机）/ 768px（平板）/ 1024px / 1440px |
| 爬虫合规 | 尊重 robots.txt，请求间隔随机化，带 User-Agent 轮换 |
| AI 防误判 | 置信度低于阈值时标记为待确认，不触发通知 |
| 性能 | 前端首屏 < 2s，WebSocket 延迟 < 100ms |

---

## 四、数据来源

| 数据源 | 类型 | 频率限制 | 需要 Key |
|--------|------|---------|---------|
| Twitter/X (twitterapi.io) | REST API | 按需调用 | 是（付费） |
| Google News RSS | RSS 爬取 | ≥30s 间隔 | 否 |
| Bing News | HTML 爬取 | ≥60s 间隔 | 否 |
| Hacker News | 官方 Algolia API | 无限制 | 否 |
| Reddit | JSON API（无需登录）| ≥2s 间隔 | 否 |
| GitHub Trending | HTML 爬取 | ≥5min 间隔 | 否 |
| arXiv | RSS 订阅 | 每日一次 | 否 |
| 技术博客 RSS | RSS 爬取（TechCrunch/36kr 等）| 每15min | 否 |
